Problem Statement:
The Challenge
In today's digital world, millions of retail investors make investment decisions based on social media, YouTube videos, online forums, and unverified recommendations without understanding the actual reasons behind those suggestions. Most stock analysis platforms display complex technical indicators such as RSI, MACD, Moving Averages, and Volume Trends, which are difficult for beginners to interpret. At the same time, many AI-powered investment tools simply provide BUY, SELL, or HOLD recommendations without explaining the reasoning behind those decisions.

This lack of transparency creates several challenges:

Investors blindly trust recommendations without understanding the underlying market signals.
Beginners struggle to interpret technical indicators and financial terminology.
Users spend significant time reading multiple financial news articles from different sources.
Existing platforms rarely combine technical analysis, market sentiment, and AI explanations into a single transparent system.
Investors find it difficult to compare multiple stocks and evaluate both opportunities and risks before investing.

As a result, many investment decisions are made without sufficient understanding, increasing the risk of financial losses.

Proposed Solution:
StockSentinel – Explainable AI Investment Assistant

StockSentinel is an Explainable AI-powered investment assistant that helps investors make informed decisions by transforming complex financial data into simple, transparent, and actionable insights.

The platform integrates real-time stock market data, technical indicators, news sentiment analysis, and Generative AI to generate recommendations that users can understand and trust.

Instead of displaying only BUY, SELL, or HOLD, StockSentinel provides:

A recommendation supported by technical analysis.
A clear explanation of why the recommendation was generated.
Confidence score and risk assessment.
AI-powered summary of recent financial news.
Side-by-side comparison between multiple stocks.
Beginner-friendly explanations of technical indicators.
A Bull vs Bear AI Debate, allowing users to understand both positive and negative perspectives before making an investment decision.

By combining Explainable AI with real-time financial analytics, StockSentinel promotes transparency, financial literacy, and responsible investing, helping users make smarter investment decisions with confidence.

 Our Goal

Our goal is not to replace financial advisors or predict the future of the market, but to build an intelligent decision-support system that helps investors understand why a recommendation is made.

StockSentinel encourages users to learn, analyze, compare, and decide, rather than blindly following AI-generated investment suggestions.

The user searches for a stock in the search bar (e.g. Infosys, TCS, or Reliance).

The user opens the StockSentinel dashboard and searches for a stock.

The frontend sends the request to the backend via REST APIs.

The selected stock symbol is sent for processing.

The backend connects to Yahoo Finance (yfinance) to fetch market data for the requested stock.

This data is used to calculate technical indicators.

Installation Guide

Follow the steps below to set up and run StockSentinel on your local machine.

📋 Prerequisites

Ensure the following software is installed before running the project:

Python 3.10 or above
Node.js (v18 or above)
npm or yarn
Git
Google Chrome (Recommended)
Visual Studio Code (Recommended)
1️⃣ Clone the Repository
git clone https://github.com/<your-username>/StockSentinel.git

Navigate to the project directory.

cd StockSentinel
2️⃣ Install Backend Dependencies

Navigate to the backend folder.

cd backend

Install the required Python libraries.

pip install -r requirements.txt

If requirements.txt is not available, install the required packages manually.

pip install flask
pip install yfinance
pip install pandas
pip install numpy
pip install ta
pip install requests
pip install python-dotenv
pip install google-generativeai
pip install flask-cors
3️⃣ Configure Environment Variables

Create a .env file inside the backend folder.

GEMINI_API_KEY=YOUR_GEMINI_API_KEY
NEWS_API_KEY=YOUR_NEWS_API_KEY

Replace the values with your own API keys.

4️⃣ Start the Backend Server
python app.py

The backend server will start at

http://localhost:5000
5️⃣ Install Frontend Dependencies

Open another terminal.

Navigate to the frontend folder.

cd frontend

Install required packages.

npm install

or

yarn install
6️⃣ Start the Frontend
npm start

or

npm run dev

The application will be available at

http://localhost:3000

or

http://localhost:5173

(depending on the frontend framework)

7️⃣ Open the Application

Open your browser and visit

http://localhost:3000

You should now see the StockSentinel Home Dashboard.

Security Measures:

StockSentinel is designed with security and reliability in mind to protect user data, API credentials, and application integrity.

1. Secure API Key Management
API keys (Gemini API, News API) are stored securely using environment variables (.env).
Sensitive credentials are never hardcoded into the source code.
The .env file is excluded from version control using .gitignore.
2. Input Validation
All user inputs are validated before processing.
Invalid stock symbols or malformed requests are handled gracefully.
Input sanitization helps prevent malicious requests and unexpected application behavior.
3. Secure API Communication
Communication between the frontend and backend is performed through REST APIs.
All external API requests use HTTPS to ensure secure data transmission.
4. Error Handling
The application implements proper exception handling to prevent crashes.
User-friendly error messages are displayed instead of exposing internal system details.
Backend errors are logged for debugging while protecting sensitive information.
5. API Rate Limiting
The system minimizes unnecessary API requests through efficient request handling.
Helps prevent excessive API usage and reduces the risk of rate-limit violations.
6. Data Privacy
StockSentinel does not collect or store personal financial information.
Market analysis is performed using publicly available stock market data.
No sensitive user data is shared with third-party services.
7. Secure Dependency Management
Only trusted and actively maintained open-source libraries are used.
Project dependencies are managed through package.json and requirements.txt.
8. Cross-Origin Resource Sharing (CORS)
Backend APIs are configured with appropriate CORS policies to allow secure communication between the frontend and backend.
9. Version Control Security
Sensitive configuration files are excluded using .gitignore.
Source code is managed using GitHub with meaningful commit history and version tracking.
10. Future Security Enhancements

The following security features can be added in future versions:

User Authentication (JWT/OAuth 2.0)
Role-Based Access Control (RBAC)
Two-Factor Authentication (2FA)
Encrypted User Portfolio Storage
API Request Authentication
Activity Logging and Audit Trails

Challenges Faced

During the development of StockSentinel, our team encountered several technical and implementation challenges while building an Explainable AI-powered investment assistant within the limited hackathon timeframe.

1. Real-Time Market Data Integration

Integrating live and historical stock market data from Yahoo Finance required handling different stock symbols, API responses, and network delays while ensuring consistent and accurate data retrieval.

2. Technical Indicator Computation

Calculating technical indicators such as RSI, MACD, Moving Averages, and Volume Trends required careful preprocessing of historical market data to generate meaningful and reliable investment insights.

3. Explainable AI Recommendation

One of the biggest challenges was designing an AI system that not only generated investment recommendations but also explained the reasoning behind them in a simple, human-readable format.

4. News Sentiment Analysis

Collecting relevant financial news and converting multiple articles into concise positive, neutral, or negative summaries while maintaining context was a challenging task.

5. Integrating Multiple Components

Synchronizing the frontend, backend, market data APIs, AI models, and technical analysis modules into a seamless workflow required effective teamwork and continuous integration.

6. Time Constraints

Developing a complete end-to-end solution within a 24-hour hackathon required careful task allocation, parallel development, and efficient project management.

7. User-Friendly Financial Explanations

Presenting complex financial indicators in a way that beginners could easily understand without oversimplifying the analysis required thoughtful UI design and prompt engineering.
